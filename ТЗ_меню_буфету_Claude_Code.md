# ТЗ: Веб-додаток «Меню шкільного буфету» — Private Boiko School (Харків)

## 1. Контекст і мета проєкту

Створи веб-додаток для шкільного буфету в Private Boiko School (м. Харків), який складається з двох частин:

1. **Публічний екран-плеєр** — крутиться на великому телевізорі в буфеті, показує меню для відвідувачів у вигляді автоматичної ротації слайдів.
2. **Адмін-панель** — завідувач буфету керує контентом (страви, ціни, категорії, фото) з будь-якого пристрою через інтернет.

Обидві частини — одна веб-сторінка з різними маршрутами. Інтернет постійний, тож робимо хмарне рішення.

---

## 2. Цільові користувачі

- **Відвідувачі** (учні, працівники школи) — пасивно дивляться на екран телевізора. Не взаємодіють.
- **Адміністратор** — **один акаунт** (завідувач буфету). Логіниться через логін+пароль, керує меню з ноутбука/планшета/телефона.

---

## 3. Технічний стек (рекомендований)

Можеш змінити на еквіваленти, але збережи простоту й хмарну сумісність:

- **Framework**: Next.js 14+ (App Router) з TypeScript
- **БД**: PostgreSQL через Prisma ORM (для production), SQLite для локальної розробки
- **Зберігання фото**: Vercel Blob, UploadThing, Cloudinary або локально `/public/uploads`
- **Стилі**: Tailwind CSS
- **Auth**: NextAuth.js (Credentials Provider) або власна реалізація на cookies+JWT
- **Drag-and-drop**: `@dnd-kit/core` + `@dnd-kit/sortable`
- **Деплой**: Vercel (для додатку) + Neon або Supabase (для PostgreSQL)

## 4. Структура даних (Prisma schema)

```prisma
model Category {
  id        String   @id @default(cuid())
  name      String   // напр. "Гарячі страви"
  order     Int      // для drag-and-drop сортування
  dishes    Dish[]
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

model Dish {
  id             String   @id @default(cuid())
  name           String
  price          Float    // у гривнях
  categoryId     String
  category       Category @relation(fields: [categoryId], references: [id])
  photoUrl       String?  // опційно
  isTopPosition  Boolean  @default(false)
  isActive       Boolean  @default(true)
  order          Int      // у межах категорії
  topOrder       Int      @default(0) // серед топ-позицій
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
}

model Settings {
  id                      String @id @default(cuid())
  categorySlideDuration   Int    @default(10)  // секунди
  topSlideDuration        Int    @default(7)   // секунди
  fadeDuration            Int    @default(500) // мс
  itemsPerCategorySlide   Int    @default(8)   // макс. страв на слайд категорії перед розбиттям на 1/2, 2/2
  updatedAt               DateTime @updatedAt
}

model User {
  id           String @id @default(cuid())
  username     String @unique
  passwordHash String
}
```

Створи seed-скрипт з:
- 1 адміном (логін/пароль через env-змінні `ADMIN_USERNAME`, `ADMIN_PASSWORD`)
- 3-4 категоріями (наприклад: «Гарячі страви», «Випічка», «Десерти», «Напої»)
- 15-20 тестовими стравами, 5-6 з них з `isTopPosition=true`
- 1 запис Settings з дефолтами

## 5. Сторінка `/display` — Екран-плеєр

Відкривається в браузері Smart TV (FullHD 1920×1080). **Безкінечний цикл показу** без жодних кнопок і інтерактивності.

### Логіка циклу

```
[Слайд категорії 1] → [Слайд категорії 2] → ... → [Слайд категорії N]
  → [Слайд топ-позицій 1] → [Слайд топ-позицій 2] → ... 
  → повтор
```

- Переходи між слайдами: **fade** (плавне затухання), тривалість з `Settings.fadeDuration`.
- Тривалість одного слайда категорії: `Settings.categorySlideDuration` (default 10 сек).
- Тривалість одного слайда топ-позицій: `Settings.topSlideDuration` (default 7 сек).

### Слайд категорії

Стиль натхненний меню кав'ярні «Клава»: **статична таблиця, чітке вирівнювання назва ↔ ціна**.

**Вигляд:**
- **Шапка зверху** (тонка горизонтальна смуга, ~10% висоти):
  - Зліва: логотип Private Boiko School (`/public/logo.png`)
  - По центру або справа: «Private Boiko School» або просто «МЕНЮ БУФЕТУ»
  - Фон шапки: чорний `#1A1A1A`, текст білий
- **Основна область** (~85% висоти):
  - Заголовок категорії великими літерами по центру (наприклад «🍽 ГАРЯЧІ СТРАВИ 🍽» або в стилізованій рамці як у «Клави»)
  - Список страв табличним розташуванням:
    - Зліва: **зарезервоване місце ~100×100px під фото-мініатюру** (показується, якщо `photoUrl` є, інакше пусто — без плейсхолдера)
    - Посередині: назва страви великим шрифтом (~40-48px)
    - Справа: ціна у форматі «105 ₴» жирним шрифтом (~44-52px)
  - Жовтий або кремовий фон (для відрізнення від топ-позицій)

**Розбиття на кілька слайдів:**
- Якщо у категорії більше ніж `itemsPerCategorySlide` активних страв — розбити на «(1/2)», «(2/2)» автоматично.
- У заголовку слайда показувати «ГАРЯЧІ СТРАВИ (1/2)».

### Слайд топ-позицій (3 страви в ряд)

**Точно у фірмовому стилі:**
- **Жовтий фон на весь екран**: `#F8C300`
- **3 вертикальні картки в ряд** з рівними відступами (по 1/3 ширини)
- Кожна картка:
  - **Чорні L-куточки** у 4 кутах картки (товщина 16px, довжина ~80px, колір `#1A1A1A`)
  - **Фото страви** зверху (займає ~40% висоти картки):
    - При завантаженні в адмінці автоматично робити чорний фон фото прозорим (поріг RGB < 30)
    - Або застосовувати CSS `mix-blend-mode: multiply`
  - **Назва страви** посередині, великими чорними літерами, до 2 рядків, `font-weight: 900`, sentence або uppercase — на твій вибір (рекомендую uppercase для впізнаваності)
  - **Червоне коло знизу**: фон `#DC1F26`, у ньому ціна білими цифрами (~56-64px) і нижче «грн» (~22-24px)
- Якщо топ-позицій менше 3 на останньому слайді — заповнити порожні місця або відцентрувати 1-2 картки.

**Зразок візуальний** (для розуміння):
```
┌─────────────────────────────────────────────┐
│  ┌──┐   ┌──┐   ┌──┐    ← чорні L-куточки    │
│  │🍪│   │🥧│   │🍰│   ← фото страв         │
│                                              │
│  КУКІЗ   ТАРТ    ТАРТ                       │
│ з фіст.  «Тропік» «Шок.вишня»                │
│                                              │
│   ⬤      ⬤      ⬤      ← червоні кола     │
│  65грн   95грн  110грн                       │
│  └──┘   └──┘   └──┘                          │
└─────────────────────────────────────────────┘
```

### Граничні випадки

1. **Немає жодної активної страви взагалі** → показувати дефолтний слайд:
   - Логотип школи по центру
   - Напис «Меню оновлюється»
   - Фон жовтий або чорний
2. **Категорія без активних страв** → пропустити в циклі (не показувати).
3. **Немає жодної топ-позиції** → блок топ-слайдів пропускається, цикл лише по категоріях.
4. **Зник інтернет** → показувати останнє кешоване меню. Реалізація: зберігати дані з `/api/display` в `localStorage` після кожного успішного запиту; при помилці мережі — читати з кешу. Опційно — Service Worker для більш надійного кешування.

### Технічні вимоги до сторінки `/display`

- Дизайн **тільки під 1920×1080** (горизонтальна орієнтація). Адаптивність не потрібна.
- **Polling**: запит `/api/display` кожні 30 секунд для отримання оновлень. Зміни підхоплюються без перезавантаження сторінки.
- **Без курсора, без скролу** — режим kiosk. Додати CSS `cursor: none; overflow: hidden`.
- **Без хедера/меню/футера** — тільки контент слайда на повний екран.
- **Прелоадинг фото** — щоб не було «мерехтіння» при переходах.

## 6. Сторінка `/admin` — Адмін-панель

### `/admin/login`
- Форма: username + password
- Один акаунт (з seed)
- Після успішного логіну — редірект на `/admin/dishes` (головна сторінка адмінки)
- Сесія через cookies (httpOnly, secure)

### Структура адмінки (sidebar навігація)

1. **Страви** (`/admin/dishes`) — головна
2. **Категорії** (`/admin/categories`)
3. **Топ-позиції** (`/admin/top-positions`)
4. **Налаштування** (`/admin/settings`)
5. **Вийти**

### `/admin/categories` — Категорії

- Список усіх категорій у вигляді **drag-and-drop списку** (`@dnd-kit/sortable`)
- Перетягування миттєво зберігає новий порядок (PUT `/api/categories/reorder`)
- Кнопка «+ Додати категорію» → модалка з полем `name`
- Для кожної категорії: іконка олівця (редагувати), іконка кошика (видалити з confirm-діалогом)
- При видаленні категорії — застереження якщо в ній є страви (запропонувати перенести або заборонити)

### `/admin/dishes` — Страви

- Список усіх страв, **згрупований за категоріями**, у кожній групі — drag-and-drop сортування
- Фільтр зверху: випадаючий список «Категорія» (всі / конкретна)
- Пошук за назвою
- Для кожної страви — рядок:
  - Мініатюра фото (або заглушка)
  - Назва
  - Ціна
  - Бейдж «ТОП» якщо `isTopPosition`
  - Toggle активності (увімкнено/вимкнено) — миттєве збереження
  - Іконки: редагувати, видалити
- Кнопка «+ Додати страву» → форма з полями:
  - **Назва** (text, required, max 100 символів)
  - **Ціна, ₴** (number, required, > 0)
  - **Категорія** (select з усіх існуючих категорій, required)
  - **Фото** (file upload, опційно):
    - Прийняти JPG, PNG, WebP
    - Превʼю після завантаження
    - Автоматичне зменшення до 800px по довшій стороні
    - Автоматичне видалення чорного фону (RGB threshold < 30) → зберегти як PNG з прозорістю
  - **Топ-позиція** (checkbox)
  - **Активна** (toggle, default увімкнено)

### `/admin/top-positions` — Топ-позиції

- Список усіх страв з `isTopPosition=true` у вигляді карток-превʼю
- **Drag-and-drop сортування** — задає порядок слайдів на екрані-плеєрі
- Картки показують той самий вигляд, що й на екрані (зменшено) — щоб адмін розумів, як це виглядатиме
- Кнопка «Перейти до страв» для редагування

### `/admin/settings` — Налаштування

Форма з полями:
- Тривалість слайда категорії, сек (number, 5-60)
- Тривалість слайда топ-позиції, сек (number, 5-60)
- Тривалість анімації переходу, мс (number, 200-2000)
- Кількість страв на одному слайді категорії (number, 4-15)
- Кнопка «Зберегти»
- Кнопка «Попередній перегляд» → відкриває `/display` у новій вкладці

### UX-принципи адмінки

- Інтерфейс **повністю українською**
- Тулбар з кнопками «Зберегти», «Скасувати»
- **Toast-повідомлення** про успіх/помилку дій
- **Confirm-діалоги** перед видаленням
- **Responsive**: працює на мобільному (адмін може редагувати з телефона)
- Лоадери при асинхронних операціях

## 7. API Endpoints

**Публічні:**
- `GET /api/display` — повертає JSON для рендеру екрану:
  ```json
  {
    "categories": [
      {
        "id": "...",
        "name": "Гарячі страви",
        "dishes": [
          { "id": "...", "name": "...", "price": 105, "photoUrl": "..." }
        ]
      }
    ],
    "topPositions": [
      { "id": "...", "name": "...", "price": 65, "photoUrl": "..." }
    ],
    "settings": {
      "categorySlideDuration": 10,
      "topSlideDuration": 7,
      "fadeDuration": 500,
      "itemsPerCategorySlide": 8
    }
  }
  ```
  Повертає тільки `isActive=true` страви, тільки `isActive` страви з категорій, відсортовані за `order`/`topOrder`.

**Захищені (middleware перевіряє сесію):**
- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET    /api/categories`
- `POST   /api/categories`
- `PUT    /api/categories/[id]`
- `DELETE /api/categories/[id]`
- `PUT    /api/categories/reorder` — body: `[{id, order}, ...]`
- `GET    /api/dishes`
- `POST   /api/dishes`
- `PUT    /api/dishes/[id]`
- `DELETE /api/dishes/[id]`
- `PUT    /api/dishes/reorder` — body: `[{id, order}, ...]` (у межах однієї категорії)
- `PUT    /api/dishes/reorder-top` — body: `[{id, topOrder}, ...]`
- `POST   /api/upload` — multipart/form-data, повертає `{ url }`
- `GET    /api/settings`
- `PUT    /api/settings`

`middleware.ts` блокує доступ до `/admin/*` і `/api/*` (крім `/api/display` і `/api/auth/login`) без авторизації.

Помістити логотип Private Boiko School у `/public/logo.png` (білий вензельний герб на чорному фоні). У шапці категорійних слайдів — використовується інверсно або у фірмових кольорах.

## 10. Кроки реалізації

1. **Setup**: створити Next.js проєкт з TypeScript, Tailwind, Prisma
2. **Schema + seed**: налаштувати Prisma schema, створити seed з тестовими даними
3. **Auth**: реалізувати логін/logout, middleware для захищених маршрутів
4. **API**: створити всі endpoints для CRUD + reorder + upload
5. **Image processing**: реалізувати обробку фото (зменшення + видалення чорного фону) на сервері при upload (використай `sharp`)
6. **Адмін-панель**:
   - Layout з sidebar
   - Сторінка категорій з drag-and-drop
   - Сторінка страв з фільтрами, формою, drag-and-drop
   - Сторінка топ-позицій
   - Сторінка налаштувань
7. **Екран-плеєр**:
   - Компонент `DisplayLoop` з логікою циклу
   - `CategorySlide` — стилізований під «Клаву»
   - `TopPositionSlide` — у фірмовому жовтому стилі з чорними куточками і червоними колами
   - `DefaultSlide` — заглушка
   - Fade-переходи через CSS або Framer Motion
   - Polling кожні 30 сек
   - Кешування в localStorage для offline
8. **Тестування** на FullHD: відкрити `/display` у вікні 1920×1080 і перевірити всі сценарії
9. **Deployment**: 
   - Створити проєкт на Vercel
   - Підключити Neon/Supabase PostgreSQL
   - Налаштувати env: `DATABASE_URL`, `ADMIN_USERNAME`, `ADMIN_PASSWORD`, `NEXTAUTH_SECRET` тощо
   - Запустити міграції і seed на production

## 11. Додаткові вимоги і нюанси

- **Безпека**: пароль адміна тільки через env-змінні. Жодних `admin/admin` за замовчуванням. Використати `bcrypt` для hash.
- **Валідація**: усі форми мають клієнтську + серверну валідацію (Zod рекомендую).
- **Помилки**: API повертає зрозумілі повідомлення українською (`{ error: "Назва не може бути порожньою" }`).
- **TypeScript**: суворі типи, без `any`.
- **Локалізація чисел**: ціни форматувати як «105 ₴» (символ гривні, пробіл).
- **Accessibility**: для адмін-панелі — alt-теги на зображення, aria-labels на кнопки. Для екрану-плеєра не критично.
- **README.md**: створити з інструкцією локального запуску, налаштування env, деплою.

## 12. Що НЕ робимо в цій версії

Для уникнення зайвої складності, в MVP **не реалізовуємо**:
- Замовлення/оплату через додаток
- Кілька ролей адмінів
- Опис страв, склад, алергени, калорійність, вагу
- Аналітику переглядів
- Сповіщення/email
- Інтеграцію з електронним щоденником або POS-системою

Це може з'явитися в наступних версіях.

## Інструкції з виконання

Реалізуй проєкт крок за кроком згідно з планом у розділі 10. Якщо щось у ТЗ неоднозначне — використовуй розумні дефолти і коментуй у коді, що саме обрав. Після реалізації:

1. Запусти проєкт локально, відкрий `/display` і перевір цикл.
2. Залогінься в `/admin`, додай кілька страв, перевір, що екран оновлюється.
3. Перевір drag-and-drop, активацію/деактивацію, видалення.
4. Створи README з інструкціями для деплою.
